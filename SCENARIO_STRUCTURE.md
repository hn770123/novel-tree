# シナリオ構造図

## 概要
「目覚めの部屋」は、プレイヤーが自分自身の深層心理と向き合うセルフセラピー型ノベルゲームです。
プレイヤーの選択によって分岐し、最終的には気持ちの良い目覚めで終わります。

## ストーリーの流れ

### 第1部：出会い（分岐なし）
```
opening（目覚め）
    ↓
meet_therapist（セラピストとの出会い）
    ↓
therapist_greeting（挨拶）
    ↓
first_question_intro（質問の開始）
```

### 第2部：心の色（第1の分岐）
```
question_color（質問：心に浮かぶ色は？）
    ├─→ [暖かいオレンジ色] → response_color_warm
    ├─→ [冷たい青色] → response_color_cool
    ├─→ [優しい緑色] → response_color_nature
    └─→ [何も思い浮かばない] → response_color_none
         ↓（すべて合流）
    question_place へ
```

**分岐の意味：**
- オレンジ色：温かさ、人との繋がりを求める心
- 青色：静けさ、自分と向き合いたい心
- 緑色：バランス、調和を求める心
- 無回答：自分の気持ちが掴めない状態

### 第3部：行きたい場所（第2の分岐）
```
question_place（質問：どんな場所に行きたい？）
    ├─→ [静かな図書館] → response_place_library → memory_trigger_book
    ├─→ [賑やかなお祭り] → response_place_festival → memory_trigger_festival
    ├─→ [広い海辺] → response_place_sea → memory_trigger_sea
    └─→ [小さな部屋] → response_place_room → memory_trigger_room
         ↓（すべて合流）
    question_feeling へ
```

**分岐の意味：**
- 図書館：内向的、静寂を求める
- お祭り：外向的、活気を求める
- 海辺：自由を求める
- 小さな部屋：安全を求める

**記憶のトリガー（間接的な過去の想起）：**
- 図書館 → 本の匂い、読み聞かせの記憶
- お祭り → 賑やかな音、誰かとの楽しい記憶
- 海辺 → 波の音、自由に走った記憶
- 部屋 → 布団の感触、安心できた記憶

### 第4部：感情の確認（第3の分岐）
```
question_feeling（質問：その記憶にどんな気持ち？）
    ├─→ [温かく、穏やかな気持ち] → response_feeling_warm
    ├─→ [少し切ない気持ち] → response_feeling_nostalgic
    ├─→ [複雑な気持ち] → response_feeling_complex
    └─→ [よくわからない] → response_feeling_unclear
         ↓（すべて合流）
    deep_question へ
```

**分岐の意味：**
- 温かい：ポジティブな記憶として受け入れている
- 切ない：過去を懐かしんでいる
- 複雑：多面的に物事を捉えている
- よくわからない：自分の感情が整理できていない

### 第5部：大切なもの（第4の分岐・エンディング分岐）
```
deep_question（質問：一番大切にしたいものは？）
    ├─→ [自分自身の心] → response_deep_self → revelation_self
    ├─→ [大切な人との繋がり] → response_deep_connection → revelation_connection
    ├─→ [自由に生きること] → response_deep_freedom → revelation_freedom
    └─→ [穏やかな日々] → response_deep_peace → revelation_peace
         ↓（すべて合流）
    awakening → ending
```

**エンディングの違い：**
1. **自己愛エンディング**：自分を大切にすることに気づく
2. **繋がりエンディング**：人との絆の大切さに気づく
3. **自由エンディング**：自分らしく生きる権利に気づく
4. **平和エンディング**：穏やかな日々の尊さに気づく

### 第6部：目覚め（共通エンディング）
```
awakening（目覚めの時）
    ↓
ending（気持ちの良い朝）
```

すべての分岐は、最終的には「気持ちの良い目覚め」という同じゴールに到達します。

## 総分岐数

- **第1分岐点（色）**：4パターン
- **第2分岐点（場所）**：4パターン
- **第3分岐点（感情）**：4パターン
- **第4分岐点（大切なもの）**：4パターン

**理論上の組み合わせ**：4 × 4 × 4 × 4 = 256通りの体験パターン
**エンディングの種類**：4種類（すべて気持ちの良い目覚め）

## シナリオの特徴

### 1. 直接的な過去の質問を避ける設計
- 「子供の頃の思い出は？」といった直接的な質問はしない
- 「心に浮かぶ色」「行きたい場所」など、抽象的な質問を通じて間接的に深層心理を探る
- プレイヤーの選択によって、自然に過去の記憶が蘇る仕組み

### 2. セルフセラピー要素
- すべての選択肢に正解・不正解はない
- セラピストは否定せず、受容的な態度で応答
- プレイヤー自身が自分の心と向き合う時間を提供

### 3. ポジティブな結末
- どの選択をしても、最終的には「気持ちの良い目覚め」に到達
- プレイヤーに希望と前向きな気持ちを残す

### 4. 象徴的な演出
- 真っ白な空間：心の中、純粋な状態
- 黒髪黒服の眼鏡の男性：自分自身を映す鏡、感情を持たない案内人
- 光が差し込む部屋：現実への回帰、新しい一日の始まり

## ノードの種類

### story（ストーリーノード）
- ナレーションや状況説明
- 自動的に次のノードへ進む

### dialogue（会話ノード）
- セラピストの台詞
- 自動的に次のノードへ進む

### choice（選択ノード）
- プレイヤーが選択肢を選ぶ
- 選択によって次のノードが変わる
- 各選択肢にはフラグが設定され、プレイヤーの傾向を記録

## 背景・キャラクター素材

### 必要な背景画像
- `white_room`：真っ白な空間
- `memory_blur`：ぼやけた記憶のイメージ
- `white_light`：光に包まれる空間
- `morning_room`：朝の光が差し込む部屋

### 必要なキャラクター画像
- `therapist_neutral`：無表情のセラピスト
- `therapist_smile`：微笑むセラピスト
- `therapist_gentle`：優しい表情のセラピスト

## プレイ時間

推定プレイ時間：10〜15分

## 使用方法

このシナリオファイルは、ノベルゲームエンジンで読み込んで使用することを想定しています。
各ノードには、テキスト、背景、キャラクター、次のノードの情報が含まれています。

### ノード構造の例
```json
{
  "id": "ノードID",
  "type": "ノードタイプ（story/dialogue/choice）",
  "speaker": "話者名",
  "text": "表示するテキスト",
  "background": "背景画像ID",
  "character": "キャラクター画像ID（オプション）",
  "next": "次のノードID（choiceタイプの場合は不要）",
  "choices": [
    {
      "text": "選択肢のテキスト",
      "next": "選択後の次のノードID",
      "flag": "設定するフラグ"
    }
  ]
}
```
